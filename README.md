## 📖 О проекте

Интерактивная пошаговая визуализация классической логической загадки из бассейна **Школы 21**.

### 🧩 Загадка

> В финале бассейна тебе нужно отправить решающий проект через один из двух терминалов:
>
> 👉 **«Счастливый»** → ведёт к успешной сдаче  
> 👉 **«Провальный»** → ведёт к автоматической несдаче
>
> Рядом с каждым терминалом — по одному пиру. Оба знают, куда ведут терминалы.  
> Но один пир **всегда говорит правду**, а другой **всегда лжёт**.  
> Кто из них кто — ты не знаешь.
>
> **❓ Какой вопрос задать, чтобы найти «счастливый» терминал?**

### ✨ Возможности

| Фича | Описание |
|-------|----------|
| 🖥️ **Хакерский терминал** | CRT-эффект, scanlines, Matrix-дождь |
| 🦆 **Пиксельная уточка** | Анимированный помощник с облачками текста |
| 💻 **Код на C** | Каждый шаг сопровождается кодом с подсветкой синтаксиса |
| 🎮 **Интерактивность** | Кнопки, квизы, выбор ответов, обратная связь |
| 📱 **Адаптивность** | Работает на мобильных и десктопе |
| 🎬 **Анимации** | Плавные переходы, typing-эффект, конфетти |

## 🌐 Деплой на GitHub Pages

Проект автоматически деплоится на GitHub Pages с помощью **GitHub Actions**.  
Ручной деплой не используется и не требуется.

---

### 🚀 Как работает деплой

В репозитории настроен workflow:

```yaml
.github/workflows/deploy.yml
```

Workflow запускается **автоматически** при `push` в ветку `main`.

**GitHub Actions:**
1. Устанавливает зависимости
2. Собирает проект (`npm run build`)
3. Публикует содержимое папки `dist/` в GitHub Pages

---

### 📦 Как задеплоить изменения

1. Внеси изменения в код
2. Зафиксируй их в Git:

```bash
git add .
git commit -m "описание изменений"
git push
```

3. После `push`:
   - GitHub Actions автоматически запустится
   - Сайт будет пересобран
   - Обновления появятся на GitHub Pages

---

### 📊 Где проверить статус деплоя

1. Открой вкладку **Actions** в репозитории
2. Найди последний workflow деплоя
3. Убедись, что он завершился со статусом ✅

---

### ⚙️ Настройка GitHub Pages (один раз)

1. Перейди в репозиторий на GitHub
2. Открой: **Settings → Pages**
3. В разделе **Source** выбери: **GitHub Actions**

> После этого сайт будет обновляться автоматически при каждом `push` в ветку `main`.

## 🎓 Ответ на загадку

<details>
<summary>⚠️ Спойлер — нажми, чтобы увидеть ответ</summary>

### Алгоритм решения:

1. Подойди к **любому** пиру
2. Спроси: **«Какой терминал, по мнению второго пира, ведёт к успешной сдаче?»**
3. Выбери **противоположный** терминал

### Почему это работает:

- **Если спросили правдивого** → он честно скажет, что лжец указал бы на *провальный* → ответ: «провальный»
- **Если спросили лжеца** → он соврёт, что правдивый указал бы на *провальный* → ответ: «провальный»

В обоих случаях ответ одинаковый! Двойная инверсия: `¬(¬x) = x` или `¬(x) = ¬x`.

Всегда выбирай **противоположный** терминал — и ты в безопасности! 🦆

</details>
